# Семейные записи студентов

Лабораторная работа №2 по курсу "Проектирование программного обеспечения интеллектуальных систем"

## Описание предметной области

Программная система предназначена для управления записями о студентах и их семьях. Каждая запись содержит информацию о:

- Студенте (ФИО)
- Отце (ФИО, заработок)
- Матери (ФИО, заработок)
- Количестве братьев и сестер

### Возможности пользователя

- Добавлять новые записи через диалоговое окно
- Просматривать все записи в главном окне с нагинацией
- Осуществлять поиск записей по различным условиям
- Удалять записи по заданным критериям с подтверждением выбора
- Сохранять и загружать данные в формате XML
- Получать сообщения о результатах операций

---

## Архитектура и описание

Проект реализует оконное приложение на базе PyQt5 с архитектурой Model-View-Controller:

| Компонент | Назначение |
|-----------|-----------|
| Model | Управление данными, валидация, работа с XML (DOM/SAX) |
| View | Графический интерфейс (главное окно и диалоги) |
| Controller | Обработка событий, координация компонентов |

### Особенности реализации

- Автосохранение данных в JSON при каждом изменении
- Поиск по целым словам в любой части ФИО (фамилия/имя/отчество)
- Множественный выбор записей для удаления
- Нагинация с настройкой количества записей на странице
- Полная валидация пользовательского ввода
- Осмысленные тестовые данные (50+ записей в каждом файле)

## Описание классов

### Класс FamilyRecord

**Файл:** `model.py`

**Назначение:** Модель данных одной записи о семье студента.

**Атрибуты:**

| Атрибут | Тип | Описание |
|---------|-----|----------|
| student_fio | str | ФИО студента |
| father_fio | str | ФИО отца |
| father_earnings | float | Заработок отца |
| mother_fio | str | ФИО матери |
| mother_earnings | float | Заработок матери |
| brothers_count | int | Количество братьев |
| sisters_count | int | Количество сестер |

**Методы:**

```python
to_dict()                   # Преобразование в словарь для сериализации
from_dict(data)             # Создание из словаря
is_empty()                  # Проверка на пустоту записи
```

### Класс  DataManager

**Файл:** `model.py`

**Назначение:** Управление коллекцией записей, сохранение/загрузка.

**Атрибуты:**

| Атрибут | Описание |
|-|-|
| records: List[FamilyRecord] | Список записей|
| data_dir: Path |Путь к папке данных |
| auto_file: Path |Путь к файлу автосохранения |


**Методы:**

```python
dd_record(record)                     # Добавление записи
search(conditions)                    # Поиск по условиям
delete_by_conditions(conditions)      # Удаление по критериям
delete_by_indices(indices)            # Удаление по индексам
save_to_xml(filepath)                 # Сохранение в XML (DOM парсер)
load_single_xml(filepath)             # Загрузка из XML (SAX парсер)
_save_auto() / _load_auto()           # Автосохранение в JSON
```

**Логика работы с XML:**
- DOM (xml.dom.minidom) — для записи: построение дерева документа вручную
- SAX (xml.sax) — для чтения: потоковая обработка, экономия памяти

### Класс  Validators

**Файл:** `view/validators.py`

**Назначение:** Валидация пользовательского ввода.

**Методы:**

```python
validate_fio(text, field_name)           # Проверка ФИО (буквы, пробелы, дефис)
validate_earnings(value, field_name)     # Проверка заработка (неотрицательный, ≤1 млрд)
validate_count(value, field_name)        # Проверка количества (0–100)
validate_search_conditions(conditions)   # Проверка заполнения условий поиска
```
## Описание представлений (View)
### MainWindow
**Файл:** `view/main_window.py`
**Назначение:** Главное окно приложения с таблицей записей.

**Компоненты:**
- QTableWidget — отображение записей (7 колонок)
- Пагинация: кнопки навигации, QSpinBox для размера страницы, счётчики
- Меню: Файл (загрузка/сохранение/выход), Записи (добавить/поиск/удалить/очистить)
- Панель инструментов с дублированием команд

**Нагинация**
- Переход: первая/последняя/предыдущая/следующая страница
- Настройка: количество записей на странице (5–50)
- Индикация: "Страница X из Y", "Всего записей: N"

### AddDialog
**Файл:** `view/add_dialog.py`
**Назначение:** Диалог добавления новой записи.

**Компоненты:**
- QLineEdit — поля для ФИО (студент, отец, мать)
- QDoubleSpinBox — поля для заработка (0–1 млрд, 2 знака после запятой)
- QSpinBox — поля для количества братьев/сестер (0–100)
- Кнопки "Сохранить" / "Отмена"
**Валидация:**
- Проверка ФИО перед сохранением
- Требование заполнить хотя бы одно поле ФИО
- Сообщения об ошибках через QMessageBox

### SearchDialog
**Файл:** `view/search_dialog.py`
**Назначение:** Диалог поиска с отображением результатов 

**Компоненты:**
- Условия поиска: ФИО (частичное совпадение по словам), заработок (диапазон), количество
- QListWidget с множественным выбором для результатов
- Пагинация результатов (аналогично главному окну)
- Кнопки: "Найти", "Выбрать все", "Снять выделение", "OK"

**Логика поиска:**
- Поиск по целым словам в любой части ФИО ("Петр" найдёт "Иванов Петр Петрович", но не "Петрович")
- Диапазон заработка: от/до (любое из значений или оба)
- Множественный выбор результатов для дальнейших действий

### DeleteDialog
  **Файл:** `view/delete_dialog.py`
  **Назначение:** Диалог удаления записей по условиям

**Компоненты:**
- Условия поиска (аналогично SearchDialog)
- Список найденных записей с чекбоксами
- Кнопки: "Найти", "Все", "Снять", "Удалить выбранные"

 **Логика работы:**
 - Ввод условий → поиск записей
 - Отображение найденных записей
 - Выбор конкретных записей для удаления
 - Подтверждение с указанием количества
 - Сообщение о результате: "Удалено записей: N"

  ### SelectionDialog
  **Файл:** `view/selection_dialog.py`

  **Назначение:** Диалог выбора при обнаружении дубликатов фамилий.

  **Компоненты:**

### Обработка исключений
**Файл:** `model.py` (встроены в методы DataManager)

Все ошибки обрабатываются через try/except с выводом сообщений в QMessageBox.

**Типы ошибок:**
- ValueError — некорректные данные записи
- FileNotFoundError — файл XML не найден
- Exception — общие ошибки с выводом детали в интерфейс

### Хранение данных
Данные сохраняются в формате XML в соответствии с требованием. Для записи используется DOM-парсер (xml.dom.minidom), позволяющий программно формировать иерархию элементов, а для чтения — SAX-парсер (xml.sax), обеспечивающий потоковую обработку и экономию оперативной памяти. Типы данных строго соответствуют семантике полей: ФИО хранятся как str, заработок — как float (с поддержкой дробной части), количество братьев/сестер — как int. Для сохранения состояния между запусками приложения реализовано фоновое автосохранение в JSON, которое работает параллельно с основным механизмом ручного экспорта/импорта XML через стандартные диалоги выбора файла.

### Принципы ООП
#### Инкапсуляция
- Приватные атрибуты в Model (префикс _ в методах _save_auto, _matches)
- Доступ к данным только через публичные методы интерфейса
#### Наследование
- Все диалоги наследуются от QDialog
- Главное окно наследуется от QMainWindow
- Валидаторы вынесены в отдельный класс для повторного использования
#### Полиморфизм
- Единый интерфейс обработки условий поиска для разных типов полей (строки, числа, диапазоны)
- Одинаковый способ отображения записей в таблице и в диалогах
#### Абстракция
- Чёткое разделение: Model (данные), View (интерфейс), Controller (логика)
- DataManager абстрагирует работу с файлами от бизнес-логики
#### Принцип единственной ответственности
- FamilyRecord — только хранение данных одной записи
- DataManager — только управление коллекцией и файлами
- Validators — только проверка корректности ввода
- Каждый диалог — только свой сценарий взаимодействия

### Сценарии демонстрации
**Сценарий 1: Добавление записи**

- Запустить приложение
- Нажать "Добавить запись" или Ctrl+N
- Заполнить ФИО студента, отца, матери
- Указать заработок и количество сиблингов
- Нажать "Сохранить"
- Убедиться, что запись появилась в таблице

**Сценарий 2: Поиск по фамилии**

- Нажать "Поиск записей" или Ctrl+F
- Ввести "Иванов" в поле "ФИО студента"
- Нажать "Найти"
- Убедиться, что найдены только записи с фамилией "Иванов" (не "Иванович")
- Отметить несколько записей, нажать "OK"
- Проверить сообщение "Выбрано: N"

**Сценарий 3: Удаление по диапазону заработка**

- Нажать "Удалить по условиям" или Del
- Указать "Заработок отца: от 50000 до 60000"
- Нажать "Найти записи для удаления"
- Отметить нужные записи из списка
- Нажать "Удалить выбранные", подтвердить
- Проверить сообщение "Удалено записей: N"
- Убедиться, что записи исчезли из главной таблицы

**Сценарий 4: Сохранение и загрузка XML**

- Добавить новую запись
- Нажать "Сохранить в XML", выбрать путь файлу
- Закрыть и перезапустить приложение
- Нажать "Загрузить из XML", выбрать нужный файл
- Убедиться, что запись появилась в таблице
